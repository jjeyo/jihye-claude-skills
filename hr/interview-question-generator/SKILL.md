# 면접 질문 생성기 (Interview Question Generator)

## 개요
이력서/포트폴리오를 분석하여 MUSINSA WAY와 Job Competency 기반의 구조화된 면접 질문셋을 생성하는 스킬입니다.

## 워크플로우

### Step 1: 입력 정보 수집

사용자에게 아래 정보를 확인합니다. 이미 제공된 정보는 다시 묻지 않습니다.

1. **이력서/포트폴리오 파일** — PDF 업로드 확인
2. **직무(Role)** — 아래 중 택1:
   - Product Designer (PD)
   - UX Researcher
   - Commerce Core PM
   - 기타 (사용자 입력)
3. **Job Level** — L3 ~ L6 (타겟 레벨 또는 현재 레벨)
4. **중점 검증 역량** — MUSINSA WAY 7가지 가치 중 이번 면접에서 특히 중점적으로 검증할 항목 (2~4개 선택)
   - 고객과 브랜드 중심
   - 오너십
   - 학습 역량
   - 결과 집중
   - 높은 기준
   - 단순화
   - 경계를 넘는 협업
5. **IC / Manager 트랙** — IC 또는 Manager (L5 이상일 경우)

> **직무별 기본 중점 역량 가이드** (사용자가 별도 선택하지 않을 경우 기본값):
> - **Product Designer**: 고객과 브랜드 중심, 높은 기준, 경계를 넘는 협업
> - **UX Researcher**: 고객과 브랜드 중심, 학습 역량, 단순화
> - **Commerce Core PM**: 결과 집중, 오너십, 경계를 넘는 협업

### Step 2: 이력서/포트폴리오 분석

업로드된 파일을 읽고 아래를 추출합니다:

- **경력 요약**: 총 경력, 주요 회사/역할, 도메인
- **핵심 프로젝트**: 규모, 역할, 성과 지표
- **기술/도구 스택**: 직무 관련 도구 및 방법론
- **강점 영역**: 반복적으로 드러나는 역량 패턴
- **검증 필요 영역**: 이력서에서 부족하거나 모호한 부분
- **포트폴리오 디테일 포인트**: 깊이 파볼 만한 프로젝트/결과물

### Step 3: 질문 생성

`references/musinsa-way.md`와 `references/job-competency.md`를 반드시 읽은 후 질문을 생성합니다.

#### 질문 구성 원칙

**50분 면접 시간 배분:**

| 구간 | 시간 | 내용 |
|------|------|------|
| 오프닝 | 3분 | 자기소개 + 아이스브레이킹 |
| 이력서/포폴 기반 질문 | 20분 | STAR 행동질문 + 포폴 디테일 (3~4개) |
| 역량 검증 질문 | 15분 | MUSINSA WAY 중점 역량 기반 (2~3개) |
| 케이스/과제형 질문 | 10분 | 가상 상황 기반 문제해결 (1~2개) |
| 마무리 | 2분 | 추가 확인 + 클로징 |

**질문 비중:**
- 이력서/포폴 기반 맞춤형: ~60%
- MUSINSA WAY 역량 기반: ~25%
- 케이스/과제형: ~15%

#### 질문 유형별 작성 가이드

**1. STAR 기반 행동질문 (이력서/포폴 기반)**
- 이력서의 구체적 프로젝트/경험을 지목하여 질문
- Situation → Task → Action → Result 구조로 답변을 유도
- 해당 레벨의 Job Competency 기대 수준과 매핑

**2. 포트폴리오 디테일 질문**
- 특정 결과물의 의사결정 과정, 트레이드오프, 데이터 근거를 파악
- "왜 이 방법을 선택했는가", "다시 한다면 어떻게 하겠는가" 관점

**3. MUSINSA WAY 역량 질문**
- 선택된 중점 검증 역량의 핵심요소(Key Elements)를 기반으로 질문 설계
- `references/musinsa-way.md`의 Just Right 행동양식을 평가 기준으로 활용
- Too Much / Too Little 행동양식을 나쁜 답변 예시의 근거로 활용

**4. 케이스/과제형 질문**
- 29CM 또는 커머스 플랫폼 맥락의 가상 상황 제시
- 직무별 맞춤 시나리오:
  - PD: 디자인 의사결정, 사용자 리서치 결과 반영, 디자인 시스템
  - UX Researcher: 리서치 설계, 인사이트 도출, 스테이크홀더 설득
  - Commerce Core PM: 지표 기반 의사결정, 우선순위 결정, 크로스팀 협업

#### 레벨별 질문 깊이 조정

**L3 (주니어):**
- 기본 역량 확인 위주, 구체적 실행 경험 중심
- "어떻게 했나요?" → 실행력, 학습 자세 검증

**L4 (미드):**
- 주도적 문제 해결, 타부서 협업 경험 중심
- "왜 그렇게 결정했나요?" → 의사결정 근거, 임팩트 검증

**L5 (시니어):**
- 조직 수준 영향력, 전략적 사고 중심
- "어떤 변화를 만들었나요?" → 리더십, 비즈니스 임팩트 검증

**L6 (리드):**
- 다조직 협업, 전사 수준 기여 중심
- "조직/회사에 어떤 기여를 했나요?" → 전략 수립, 문화 형성 검증

### Step 4: 산출물 포맷

각 질문은 아래 구조로 작성합니다:

```
## Q{번호}. [질문 유형 태그] 질문 내용
- **검증 역량**: [MUSINSA WAY 가치] + [Job Competency 축]
- **질문 의도**: 이 질문을 통해 확인하려는 것
- **좋은 답변 예시**: 이런 답변이 나오면 긍정 평가
- **나쁜 답변 예시**: 이런 답변이 나오면 부정 평가 (Too Much/Too Little 기준)
- **Follow-up 질문**: 
  - (답변이 좋을 때) 더 깊이 파는 추가 질문
  - (답변이 부족할 때) 다른 각도로 재검증하는 질문
- **예상 소요시간**: X분
```

**질문 유형 태그:**
- `[STAR]` — STAR 기반 행동질문
- `[포폴]` — 포트폴리오 디테일 질문
- `[MW:가치명]` — MUSINSA WAY 역량 질문 (ex: `[MW:오너십]`)
- `[케이스]` — 케이스/과제형 질문

### Step 5: 최종 산출물 구성

산출물은 아래 순서로 구성합니다:

1. **면접 개요**
   - 후보자 정보 요약 (이력서 기반)
   - 직무 / 레벨 / 중점 검증 역량
   - 면접 시간 배분 가이드

2. **이력서 분석 요약**
   - 경력 하이라이트
   - 강점 영역 / 검증 필요 영역
   - 포폴 디테일 포인트

3. **질문 리스트** (위 포맷에 따라 7~10개)

4. **면접 운영 팁**
   - 오프닝 멘트 예시
   - 시간 관리 가이드
   - 평가 시 유의사항

### 산출물 형식

기본적으로 **마크다운 파일(.md)**로 생성하여 `/mnt/user-data/outputs/`에 저장합니다.
파일명: `면접질문_{직무}_{레벨}_{후보자명}_{날짜}.md`

사용자가 docx나 다른 포맷을 원하면 해당 스킬을 참조하여 변환합니다.

## 참조 파일

스킬 실행 시 반드시 아래 참조 파일을 읽어야 합니다:

1. `references/musinsa-way.md` — MUSINSA WAY 7가지 가치 + 핵심요소 + 행동양식 (Just Right/Too Much/Too Little)
2. `references/job-competency.md` — 테크부문 Job Level Definition (L3~L8, IC/Manager, 5개 역량 축)

## 주의사항

- 질문은 반드시 이력서/포폴 내용을 구체적으로 언급해야 합니다 (일반적인 질문 X)
- MUSINSA WAY의 행동양식(Just Right/Too Much/Too Little)을 평가 기준에 반영해야 합니다
- Job Competency의 레벨별 기대 수준을 질문 깊이에 반영해야 합니다
- 50분이라는 제한 시간 내에 소화 가능한 분량이어야 합니다
- 케이스/과제형 질문은 29CM 또는 커머스 플랫폼 맥락이어야 합니다
- 면접은 한국어로 진행되므로 질문도 한국어로 작성합니다
