# 29Play Logo Configuration

29CM 앱의 좌측 상단 로고(29플레이)를 변경하기 위한 Amplitude JSON을 생성한다.

## 워크플로우

### 1. 필수 정보 수집

사용자에게 다음 정보를 확인:

| 항목 | 설명 | 예시 |
|------|------|------|
| 시작일시 | 로고 적용 시작 | 2026-02-16 00:00:00 |
| 종료일시 | 로고 적용 종료 | 2026-02-18 23:59:59 |
| 로고 URL (화이트) | 밝은 배경용 로고 | asset.29cm.co.kr/cms/... |
| 로고 URL (블랙) | 어두운 배경용 로고 | asset.29cm.co.kr/cms/... |
| 커스텀 탭 ID | 프로모션 시점 노출 중인 커스텀 탭 | 106 |

### 2. 탭 ID 참조

| ID | 탭 이름 | 상태 | 로고 컬러 |
|----|--------|------|----------|
| 1 | 우먼 탭 | 사용 | **화이트** |
| 2 | 홈 탭 | 사용 | **화이트** |
| 3 | 라이프 탭 | 구버전 일부 노출 | **화이트** |
| 4 | 테크 탭 | 미사용 | **화이트** |
| 5 | 뷰티 탭 | 사용 | **화이트** |
| 6 | 컬쳐 탭 | 미사용 | **화이트** |
| 7 | 푸드 탭 | 미사용 | **화이트** |
| 8 | 맨 탭 | 사용 | **화이트** |
| 9 | 베스트 탭(구버전) | 구버전 일부 노출 | **블랙** |
| 37 | 베스트 탭 | 사용 | **블랙** |
| 54 | 발견 탭 | 사용 | **블랙** |
| 커스텀 탭 | 프로모션 시 확인 필요 | 동적 | **블랙** |

### 3. 로고 컬러 규칙

**블랙 로고 적용 탭:**
- 커스텀 탭 (프로모션 시 생성된 탭 ID)
- 발견 탭 (ID: 54)
- 베스트 탭 (ID: 9, 37)

**화이트 로고 적용 탭:**
- 이외 모든 카테고리 탭

### 4. JSON 생성 템플릿

```json
{
  "data": [
    {
      "endAt": "{종료일시}",
      "tabId": {탭ID},
      "logoUrl": "{로고URL}",
      "startAt": "{시작일시}",
      "deeplink": "",
      "logoType": "tabId-{번호}"
    }
  ]
}
```

### 5. 기본 탭 구성 (현재 사용 중)

JSON 생성 시 포함해야 할 기본 탭 ID:
- 1 (우먼) - 화이트
- 2 (홈) - 화이트
- 5 (뷰티) - 화이트
- 8 (맨) - 화이트
- 37 (베스트) - 블랙
- 54 (발견) - 블랙

추가 탭 (필요시):
- 76, 84, 85, 86, 105 - 화이트
- 커스텀 탭 ID - 블랙

### 6. 출력 형식

1. **JSON 코드 블록**: Amplitude에 입력할 완성된 JSON
2. **변경 요약 테이블**: 탭별 로고 설정 내역
3. **적용 안내**: Amplitude 설정 경로 및 주의사항

## 예시 요청

사용자: "설날 29플레이 적용해줘. 2/16 00:00 ~ 2/18 23:59. 커스텀탭 106번."

→ 위 정보로 JSON 생성하고 블랙/화이트 로고 URL 확인 후 적용
