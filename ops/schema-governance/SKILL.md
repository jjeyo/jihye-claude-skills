# Schema Governance — Slack 기반 분석 MD 거버넌스

## 개요

29CM 데이터 분석에 사용되는 MD 파일(스키마, 쿼리 패턴, 지표 정의)을 팀과 공유하되,
**관리자(Jihye)만 업데이트할 수 있는** Slack 기반 피드백 워크플로우를 제공합니다.

## 워크플로우

```
┌─────────────┐     ┌──────────────┐     ┌───────────────┐     ┌──────────────┐
│  1. 공유     │ ──→ │  2. 피드백    │ ──→ │  3. 검증      │ ──→ │  4. 반영     │
│ /share-schema│     │ 팀원 스레드   │     │/review-feedback│     │/apply-feedback│
│ Slack 채널   │     │ 댓글 작성     │     │ 관리자 검토    │     │ MD 업데이트   │
└─────────────┘     └──────────────┘     └───────────────┘     └──────────────┘
      ↑                                                              │
      └──────────────── 업데이트 후 재공유 (선택) ──────────────────────┘
```

## 관리 대상 MD 파일

| 파일 | 경로 | 설명 |
|------|------|------|
| 테이블 스키마 | `~/.claude/skills/29cm-data-query/references/schema.md` | DB 테이블/컬럼 정의 |
| 쿼리 패턴 | `~/.claude/skills/29cm-data-query/references/query-patterns.md` | 분석 유형별 쿼리 예시 |
| CE 지표 정의 | `~/.claude/skills/29cm-data-query/references/ce-metrics.md` | 팀 KPI/지표 정의 |
| 스킬 가이드 | `~/.claude/skills/29cm-data-query/SKILL.md` | 쿼리 작성 가이드 |

## 커맨드

### `/share-schema` — Slack에 MD 공유
팀원이 읽을 수 있도록 분석 MD 내용을 Slack 채널에 포스팅합니다.

### `/review-feedback` — 피드백 수집 및 검토
Slack 스레드에 달린 피드백을 수집하고, 관리자가 하나씩 승인/반려합니다.

### `/apply-feedback` — 승인된 피드백 반영
검증 완료된 피드백을 실제 MD 파일에 반영합니다.

## 권한 모델

| 역할 | 공유 보기 | 피드백 작성 | 피드백 검증 | MD 업데이트 |
|------|----------|-----------|-----------|-----------|
| 관리자 (Jihye) | ✅ | ✅ | ✅ | ✅ |
| 팀원 | ✅ | ✅ | ❌ | ❌ |

## 피드백 형식 가이드

팀원이 Slack 스레드에 피드백을 남길 때 권장하는 형식:

```
📝 피드백
- 파일: schema.md (또는 query-patterns.md, ce-metrics.md)
- 위치: [섹션명 또는 테이블명]
- 현재 내용: [잘못된 현재 내용]
- 수정 제안: [올바른 내용]
- 사유: [왜 수정이 필요한지]
```
