# CE 팀 지표 정의

## 북극성 지표

**Engagement CVR** = 전환 유저 / 참여 유저

- 참여 유저: 특정 기능을 사용한 유저
- 전환 유저: 참여 후 구매까지 이어진 유저

### Success Metrics (Main)

| 지표 | 정의 | 쿼리 조건 |
|------|------|----------|
| 클릭 세션율 | 랭킹 탭 진입 세션 중 클릭 발생 세션 비율 | 분모: `view_page` + `best_main`, 분자: `click_item`/`click_category`/`click_button` + `best_main` |
| GGMV | 랭킹 탭 방문 유저의 **당일** 주문 GMV | 동일일 귀속 (`visit_date = order.dt`) |

|------|------|----------|
| PDP 진입율 | 랭킹 탭 진입 세션 중 상품 상세 진입 비율 | 세션 시퀀스: `best_main` 진입 후 같은 세션에서 `view_page` + `item_detail` |
| 브랜드홈 방문수 | 랭킹 탭 경유 브랜드홈 방문 세션 | 세션 시퀀스: `best_main` 진입 후 같은 세션에서 `view_page` + `brand_home` |
| 선물하기 구매수 | 랭킹 선물 탭 경유 선물 구매 | 2026-04-01 배포 후 분석 가능 |

### Guardrail Metrics

| 지표 | 정의 | 쿼리 조건 |
|------|------|----------|
| CVR | 랭킹 탭 방문 유저의 **당일** 구매 전환율 | 동일일 귀속 (`visit_date = order.dt`) |

### 분석 시 주의사항

1. **귀속 기준**: 유저 기반 기간 내 overlap 사용 금지 → **세션 시퀀스** 또는 **동일일 귀속** 사용
2. **기간 보정**: Before/After 일수가 다르면 반드시 **일평균** 비교
3. **비율 vs 절대값**: 유입 증가 시 비율 하락이 발생할 수 있음 (분모 희석). 반드시 일평균 절대값도 함께 확인
4. **PDP 이벤트**: `visit_item_detail` 아님 → `view_page` + `page_name = 'item_detail'`

## 카테고리/브랜드 탐색 지표

### 1. 카테고리 브랜드 탭 클릭률 (CTR)
- 정의: 카테고리 메인 페이지에서 브랜드 탭으로 이동하는 비율
- 분모: `view_page` (page_name = 'category_main')
- 분자: `click_button` (page_name = 'category_brand_index')
- 계산: UNIQUES(분자) / UNIQUES(분모) * 100
- 참고 차트: https://app.amplitude.com/analytics/style-share/chart/q4ntacfm

### 2. 지면별 상품 클릭률
- 정의: 각 탐색 지면(brand_home, category_list 등)에서의 상품 클릭 성과
- 이벤트: click_item
- 계산: 상품 클릭 수 / 세션 수
